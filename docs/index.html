DeltaForge

Quick Start
Define a pipeline in YAML. Run it.
No boilerplate code, no JVM, no Zookeeper. One config file, one binary.
pipeline.yaml
metadata: name: orders-to-kafka tenant: acme spec: source: type: mysql config: dsn: ${MYSQL_DSN} tables: [shop.orders] processors: - type: javascript inline: | // transform, filter, route function processBatch(events) { return events; } sinks: - type: kafka config: brokers: ${KAFKA_BROKERS} topic: cdc.${source.table} envelope: type: debezium
Capture
Connect to MySQL binlog or PostgreSQL logical replication. Wildcard table patterns supported.
Transform
JavaScript processors for filtering, enrichment, PII redaction, and per-event routing decisions.
Route
Template strings resolve per-event: cdc.${source.table} routes each table to its own topic.
Deliver
Fan-out to Kafka, Redis, and NATS simultaneously. Debezium, CloudEvents, or Native wire formats.
Checkpoint
At-least-once delivery with configurable commit policies. Resume exactly where you left off after restarts.
Why DeltaForge
Built for production (CDC)
Stream database changes reliably with minimal overhead.
Powered by Rust
Predictable latency, memory safety, and minimal resource footprint.
Dynamic Routing
Route events to per-table topics, streams, or subjects using template strings or JavaScript logic.
Schema Discovery
Automatically infer and track schemas from event payloads with deep inspection of nested JSON structures.
Zero-Downtime DDL
Detects schema changes and reloads automatically. No pipeline restart, no manual intervention.
Graceful Failover
Handles source failover with automatic schema revalidation. Reconnects with exponential backoff and jitter.
Reliable Checkpoints
At-least-once delivery with pluggable backends. Configurable commit policies: all, required, or quorum.
High-Cardinality Keys
Probabilistic detection of dynamic map keys prevents false schema evolution from session IDs and trace IDs.
Live Management
Pause, resume, patch, and inspect running pipelines via REST API. Prometheus metrics and health probes built in.
Debezium Compatible
Drop-in wire format compatibility. Migrating from Debezium? Switch the envelope type and you're done.
Output Formats
Three envelope formats, one config field
Match the wire format your consumers expect. Switch with a single line of YAML.
Native Lowest overhead
Minimal envelope for DeltaForge-native consumers. No wrapper overhead.

Dynamic Routing
Route every event to the right destination
Template strings for simple per-table routing. JavaScript for conditional business logic.
# Routes each table to its own topic automatically sinks: - type: kafka config: topic: cdc.${source.db}.${source.table} key: ${after.customer_id} - type: redis config: stream: events:${source.table} key: ${after.id} - type: nats config: subject: cdc.${source.db}.${source.table} # Available: ${source.table}, ${source.db}, ${op}, # ${after.<field>}, ${before.<field>}, ${tenant_id}
function processBatch(events) { for (const ev of events) { if (!ev.after) continue; if (ev.after.total_amount > 10000) { // High-value orders → priority topic ev.route({ topic: "orders.priority", key: String(ev.after.customer_id), headers: { "x-tier": "high-value", "x-amount": String(ev.after.total_amount) } }); } else { ev.route({ topic: "orders.standard", key: String(ev.after.customer_id) }); } } return events; }
